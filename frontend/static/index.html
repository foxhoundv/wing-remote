⚠ /dev/snd not detected yet — this is normal if the Wing USB cable isn't connected.

You can still enable passthrough now. The setting will be written to docker-compose.yml and activated on next container start. Once you connect the Wing via USB and restart the container, audio devices will appear here.

Common reasons /dev/snd is absent:
• Wing USB cable not yet connected to this machine
• Running on WSL1 (use WSL2 + usbipd) or macOS Docker Desktop
• ALSA drivers not loaded — try: sudo modprobe snd-usb-audio
