⚠ /dev/snd not detected yet — this is normal if the Wing USB cable isn't plugged in.

You can enable recording now anyway. The container uses privileged: true and auto-detects /dev/snd each time it starts — no docker-compose.yml edits needed. Just connect the Wing via USB and run docker compose restart wing-remote.

Common reasons /dev/snd is absent:
• Wing USB cable not yet connected
• Running on WSL1 (use WSL2 + usbipd) or macOS Docker Desktop
• ALSA drivers not loaded: sudo modprobe snd-usb-audio
